# AdOS — Ad Operating System

> **"The intelligence layer that transforms AdBazaar into a real-world performance marketing platform."**

---

## ⚠️ IMPORTANT: Build Order

**AdOS is NOT the first priority.** Building AI before data is over-engineering.

### Correct Build Order:

```
Phase 1 (NOW):      AdsQr + AdBazaar fixes + Attribution
Phase 2 (30-90 days): Lite AdOS (rule-based)
Phase 3 (90+ days):  Full AdOS (ML-based)
```

### Why?

| System | Needs |
|--------|-------|
| **AdsQr** | Minimal data, fast to launch |
| **AdBazaar fixes** | Reliability, trust |
| **Attribution** | Clean funnel, real tracking |
| **Lite AdOS** | Some data, rule-based logic |
| **Full AdOS** | 100+ campaigns, ML models |

---

## Vision

**AdOS** is not a separate product — it's the **intelligence layer** that makes AdBazaar smarter. It analyzes data across the entire ReZ ecosystem to:

1. **Predict** campaign performance before launch
2. **Optimize** budget allocation in real-time
3. **Recommend** the best placements and strategies
4. **Attribute** conversions across online/offline

### Start with Data-Backed Estimates, Not AI

> **"Data-backed estimates that improve over time"**

NOT:

> **"85-90% AI prediction accuracy"**

We can claim accuracy only after we have data to validate it.

---

## What AdOS Is NOT

- Not a new app
- Not a replacement for AdBazaar
- Not a competitor to Google/Meta

## What AdOS IS

- An **intelligence layer** that sits on top of AdBazaar
- A **recommendation engine** for campaign optimization
- A **prediction system** for ROI forecasting
- A **unified dashboard** showing all ad performance

## AdOS API Specification

### 1. Campaign Planner API

```typescript
// POST /ados/api/campaign/plan
interface CampaignPlanRequest {
  goal: 'foot_traffic' | 'brand_awareness' | 'direct_sales' | 'app_install';
  category: string;
  budget: number;
  duration_days: number;
  target_audience?: {
    age_range?: [number, number];
    income_level?: 'low' | 'mid' | 'high';
    interests?: string[];
  };
  locations?: string[]; // city IDs or zones
}

interface CampaignPlanResponse {
  recommendations: {
    locations: Array<{
      id: string;
      name: string;
      score: number; // 0-100
      estimated_scans: [number, number];
      estimated_visits: [number, number];
      budget_allocation: number;
    }>;
    ad_types: Array<{
      type: string;
      allocation: number;
      rationale: string;
    }>;
  };
  predictions: {
    total_scans: [number, number];
    total_visits: [number, number];
    total_purchases: [number, number];
    roas: [number, number];
    confidence: number; // 0-1
  };
  risks: Array<{
    factor: string;
    impact: 'low' | 'medium' | 'high';
    mitigation: string;
  }>;
}
```

### 2. Budget Optimizer API

```typescript
// POST /ados/api/campaign/:id/optimize
interface OptimizeRequest {
  current_spend: number;
  performance_data: Array<{
    location_id: string;
    spend: number;
    scans: number;
    visits: number;
    purchases: number;
  }>;
}

interface OptimizeResponse {
  recommendations: Array<{
    location_id: string;
    action: 'increase' | 'decrease' | 'pause';
    suggested_spend: number;
    reason: string;
    expected_improvement: number; // percentage
  }>;
  projected_improvement: number; // overall ROAS improvement
  risk_level: 'low' | 'medium' | 'high';
}
```

### 3. Audience Insights API

```typescript
// GET /ados/api/audience/insights
interface AudienceInsightsRequest {
  location_ids?: string[];
  category?: string;
  time_range?: '7d' | '30d' | '90d';
}

interface AudienceInsightsResponse {
  segments: Array<{
    name: string;
    size: number;
    conversion_rate: number;
    preferred_ad_types: string[];
    peak_times: string[];
  }>;
  trends: Array<{
    metric: string;
    change: number; // percentage
    direction: 'up' | 'down';
  }>;
  opportunities: Array<{
    segment: string;
    potential: number;
    recommendation: string;
  }>;
}
```

### 4. Attribution API

```typescript
// POST /ados/api/attribution/track
interface AttributionEvent {
  event_type: 'scan' | 'visit' | 'purchase';
  campaign_id: string;
  qr_id: string;
  user_id?: string;
  device_id?: string;
  timestamp: string;
  location?: {
    lat: number;
    lng: number;
    city: string;
  };
  metadata?: Record<string, unknown>;
}

// Attribution result links offline action to online conversion
interface AttributionResult {
  touchpoints: Array<{
    campaign_id: string;
    channel: string;
    weight: number; // contribution to conversion
  }>;
  attributed_revenue: number;
  attribution_model: 'first_touch' | 'last_touch' | 'linear' | 'time_decay';
}
```

---

## Attribution Models

### Supported Models

| Model | Best For | How It Works |
|-------|---------|-------------|
| **First Touch** | Awareness campaigns | 100% credit to first ad seen |
| **Last Touch** | Direct response | 100% credit to last ad seen |
| **Linear** | Balanced | Equal credit to all touchpoints |
| **Time Decay** | Recent influence | More credit to recent touchpoints |
| **Position Based** | Mixed | 40% first, 20% middle, 40% last |
