## Build Order (Critical)

### Why Order Matters

```
WRONG ORDER:
├── Build AI first
├── No real data
├── "Fake intelligence"
└── AI doesn't work

RIGHT ORDER:
├── AdsQr (fast, simple)
├── Generate real data
├── Attribution works
├── Lite AdOS (rule-based)
├── Data validates predictions
├── Full AdOS (ML)
└── Scale
```

## Phase 1: AdsQr + Attribution (NOW)

### Timeline: 4-6 weeks

### Goal: 100-500 active campaigns

### Why First?

| Reason | Explanation |
|--------|-------------|
| **Fast to launch** | 5 minutes to create campaign |
| **Low commitment** | ₹500 to start |
| **Generates data** | Real scan, visit, purchase data |
| **Validates loop** | Does attribution actually work? |
| **Attracts merchants** | SMBs can try immediately |

### Features

```
ADSQR MVP:
├── Campaign creation (simple)
├── QR code generation
├── Template landing page
├── Multi-step rewards (scan/visit/purchase)
├── Hybrid coins (REZ + Brand)
├── Basic analytics (scans, users, coins)
└── REZ wallet integration
```

### Success Criteria

| Metric | Target |
|--------|--------|
| Active campaigns | 100+ |
| Total scans | 10,000+ |
| Attribution accuracy | 80%+ |
| Merchants onboarded | 50+ |

## Phase 3: Lite AdOS (After 100+ campaigns)

### Timeline: 4-6 weeks after AdsQr

### Goal: Rule-based intelligence

### When to Start

- 100+ active campaigns
- Clean attribution data
- Enough historical data for patterns

### Features

```
LITE ADOS:
├── "Best performing locations" (rule-based)
├── "Budget allocation" suggestions
├── Simple predictions (based on category)
├── Basic recommendations
└── Dashboards
```

- ❌ 90% prediction accuracy

## Decision Framework

### Before Building Any Feature

Ask:

1. **Do we have data to support this?**
   - Yes → Build
   - No → Wait

2. **Is attribution working?**
   - Yes → Add intelligence
   - No → Fix attribution first

3. **Will this work without AI?**
   - Yes → Rule-based first
   - No → Requires ML (later)
