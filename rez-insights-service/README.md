# rez-insights-service

AI-generated insight storage and delivery service for the ReZ ecosystem. Stores AI-generated insights from ReZ Mind for user consumption and analytics.

---

## Overview

The `rez-insights-service` is responsible for storing, managing, and delivering AI-generated insights from ReZ Mind to users via the Copilot UI. It provides a centralized repository for all intelligent recommendations, predictions, and analytics generated by the AI pipeline.

## Purpose

- **Insight Storage**: Persist AI-generated insights with metadata
- **User Delivery**: Serve insights to Copilot UI for user consumption
- **Categorization**: Organize insights by type, category, and priority
- **Analytics**: Track insight engagement (read, dismissed, clicked)
- **Personalization**: Support personalized insight delivery based on user preferences

---

## Features

- Create and manage AI-generated insights
- Support for multiple insight types (recommendation, prediction, anomaly, summary, alert)
- Priority-based insight management
- Redis caching for improved performance
- Rate limiting protection
- JWT authentication
- MongoDB for persistent storage
- Event-driven architecture

## API Endpoints

### Create Insight
```
POST /api/insights
```

**Request Body**:
```json
{
  "userId": "uuid",
  "insightType": "recommendation",
  "category": "inventory",
  "title": "Low Stock Alert",
  "description": "Several items are running low",
  "content": {
    "items": [...],
    "threshold": 10
  },
  "confidenceScore": 0.92,
  "source": "inventory-service",
  "sourceId": "alert-123",
  "priority": 1
}
```

**Response** (201 Created):
```json
{
  "id": "uuid",
  "createdAt": "2026-05-01T10:00:00Z",
  "success": true
}
```

### Get User Insights
```
GET /api/insights/user/:userId
```

**Query Parameters**:
- `category` - Filter by category (e.g., `inventory`, `travel`, `spending`)
- `type` - Filter by insight type (e.g., `recommendation`, `prediction`, `anomaly`)
- `unreadOnly` - Only return unread insights (e.g., `true` or `false`)
- `limit` - Number of results (default: 50, max: 100)
- `offset` - Pagination offset

**Response** (200 OK):
```json
{
  "insights": [
    {
      "id": "uuid",
      "userId": "uuid",
      "insightType": "recommendation",
      "category": "inventory",
      "title": "Low Stock Alert",
      "description": "Several items are running low",
      "content": {...},
      "confidenceScore": 0.92,
      "priority": 1,
      "readAt": null,
      "createdAt": "2026-05-01T10:00:00Z"
    }
  ],
  "total": 125,
  "unreadCount": 15
}
```

### Get Single Insight
```
GET /api/insights/:insightId
```

### Mark Insight as Read
```
PATCH /api/insights/:insightId/read
```

### Dismiss Insight
```
PATCH /api/insights/:insightId/dismiss
```

### Bulk Mark Read
```
POST /api/insights/bulk/mark-read
```

### Health Check
```
GET /health
```

## Insight Types

| Type | Description | Example |
|------|-------------|---------|
| `recommendation` | Actionable suggestions | "Consider adding more inventory" |
| `prediction` | Forecast-based insights | "High demand expected next week" |
| `anomaly` | Unusual pattern detection | "Unusual order spike detected" |
| `summary` | Aggregated information | "Weekly spending summary ready" |
| `alert` | Important notifications | "Low stock warning for Item X" |

## Categories

| Category | Description |
|----------|-------------|
| `inventory` | Inventory and stock related |
| `travel` | Travel and hotel bookings |
| `spending` | Spending patterns and analysis |
| `loyalty` | Loyalty program insights |
| `fraud` | Fraud detection alerts |
| `general` | General purpose insights |

## Environment

| Variable | Required | Default | Description |
|----------|----------|---------|-------------|
| `INTENT_API_URL` | Yes | - | ReZ Mind API endpoint |
| `DATABASE_URL` | Yes | - | PostgreSQL connection string |
| `JWT_SECRET` | Yes | - | JWT verification secret |
| `PORT` | No | `3008` | Service port |
| `CACHE_TTL` | No | `3600` | Cache TTL in seconds |
| `MAX_INSIGHTS_PER_USER` | No | `1000` | Maximum stored insights per user |
| `LOG_LEVEL` | No | `info` | Logging level |

## Events

### Events Published

| Event | Payload | Description |
|-------|---------|-------------|
| `insight.created` | `{ insightId, userId, type }` | New insight generated |
| `insight.read` | `{ insightId, userId }` | User read insight |
| `insight.dismissed` | `{ insightId, userId }` | User dismissed insight |

### Events Consumed

| Event | Source | Action |
|-------|--------|--------|
| `intent.processed` | ReZ Mind | Store generated insights |
| `analysis.complete` | Analytics service | Store analysis results |

## Error Responses

All error responses follow this format:

```json
{
  "success": false,
  "error": "Error message describing what went wrong"
}
```

Common HTTP status codes:
- `200`: Success
- `201`: Created
- `400`: Bad Request (validation error)
- `401`: Unauthorized
- `403`: Forbidden
- `404`: Not Found
- `429`: Too Many Requests (rate limited)
- `500`: Internal Server Error
