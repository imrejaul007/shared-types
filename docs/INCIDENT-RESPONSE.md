# Incident Response Plan

## Severity Levels

| Level | Definition | Response Time |
|-------|------------|---------------|
| SEV1 | Service down | 15 min |
| SEV2 | Major feature broken | 1 hour |
| SEV3 | Minor feature broken | 4 hours |
| SEV4 | Cosmetic issue | 24 hours |

## Response Process

### 1. Detection
- Automated alerts
- User reports
- Monitoring dashboards

### 2. Triage
- Assess severity
- Identify scope
- Assign incident commander

### 3. Communication
- Notify stakeholders
- Update status page
- Slack #incidents

### 4. Resolution
- Diagnose root cause
- Implement fix
- Verify resolution

### 5. Post-Mortem
- Document timeline
- Identify improvements
- Update runbooks

## On-Call

### Rotation
- Primary: Week 1
- Secondary: Week 2
- Escalation: Week 3

### Tools
- PagerDuty
- Slack #oncall
- Runbooks
